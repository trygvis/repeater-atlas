# Better server running for agents

- Make the main application respect the PORT environment variable, default to
  the current 3000.
- If the port is "0" (meaning that the OS should allocate a port for you), check
  what the port became after creating the listening socket and print it.
- Add a command line argument to the main application to set the port via
  "--port=3000".
- Expand the "just run"
  - Set PORT to 3000 if not set
  - Launch repeater-atlas with PORT as the "--port" argument
  - Create a PID file before launching the server
  - Use `kill $(cat repeater-atlas.pid)` to terminate the correct instance of
    the repeater atlas before launching the new one. This way it is safe to run
    "just run" after changing the code directly.
- Update AGENTS.md
  - make it clear that it should start the server with PORT=3001.
  - Note that it should look at the log file (see ticket ra-0e10) to figure out
    what happened if something failed.
