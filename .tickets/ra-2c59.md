# New feature: Allow searching for lat/long on map

When searching, allow searching for a lat/long in addition to the existing
search.

The lat/long should be very flexible in its parsing, and allow the user to
specify in at least these formats:

| Input            | Latitude | Longitude |
| ---------------- | -------- | --------- |
| 12.345,-34.6565  | 12.354   | -34.6565  |
| N12.354,W34.6565 | 12.354   | -34.6565  |
| +12.354,34.6565  | 12.354   | 34.6565   |

- Make the location parser its own utility.
- Make unit tests for the parsing, and use the parser in the search feature.
  Make sure that the search component is its own feature (separate rust file)
  that sits on top of the actual SQL search and location parsing.
- Do necessary refactorings if required first.
