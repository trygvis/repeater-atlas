# New feature: Personal logbook

"My page" gets a dedicated logbook section with a form where the user configures
options and clicks a button to download the PDF. The user fills in the options
every time — nothing is persisted. The PDF is generated on the fly and streamed
directly as a download.

## Configuration form

All of the following are form fields on "my page":

- Page size: A4 or A5
- Number of log pages (default 10)
- Optional pages (all toggles, any combination allowed):
  - Title page (first): prints "Logbook for $CALL"
  - Locations page (last): list of the user's saved locations from ra-4ca0
    (UserLocation); each entry shows address, maidenhead, and lat/long
  - Phonetic alphabet page
  - Frequency bands page (IARU Region 1 / Europe, all bands HF–SHF)

## PDF layout

- Orientation: landscape
- Page header: a field to write the current date and location
- Page footer: user's call sign printed, plus a field to write page number

## Log table columns

- QSO #
- Start
- End
- Station
- Operator
- QTH
- RST sent (narrow, 3 digits)
- RST received (narrow, 3 digits)
- Freq
- Mode
- Power
- Remarks

## Non-functional requirements

- Generated with Typst. The `typst` binary must be vendored into the Docker
  image — it cannot be assumed present on the host.
- The frequency bands page must note that it covers IARU Region 1 / Europe.

## Notes

**2026-04-04T05:39:06Z**

Implemented. Changes:

- `src/service/logbook.rs`: Typst template renderer and PDF generation. Writes a
  `.typ` file to a temp dir, runs `typst compile`, returns the PDF bytes.
  `TYPST_BIN` env var overrides the binary path (default `/app/bin/typst`).
- `src/web/logbook.rs`: POST `/-/export/logbook.pdf` handler. Reads form fields,
  fetches user locations, delegates to the service, streams the PDF.
- `Dockerfile`: new `typst-vendor` stage downloads the typst binary (v0.13.1,
  x86_64-musl) and copies it to `/app/bin/typst` in the runtime image.
- My page: Logbook PDF section with page size selector, log page count, and four
  optional-page checkboxes.
- Docs: `my-page.md` (Logbook PDF section), `IMPLEMENTATION.md` (route).

**2026-04-04T16:42:21Z**

Follow-up changes after initial implementation:

- Route renamed from `/-/export/logbook.pdf` to `/-/my/logbook.pdf`; handler and
  path struct renamed accordingly.
- Logbook PDF section moved out of the Exports section into its own section on
  my page.
- `typst` is now assumed to be on PATH (no `TYPST_BIN` env var, no
  `/app/bin/typst` default). The Dockerfile copies the binary to
  `/usr/local/bin/typst`.
- Startup check: `typst --version` is run at startup and logged; failure is
  shown as a single warning.
- All optional pages are checked on by default in the form.
- `templates/logbook.typ` is now a proper Askama template. The `#logbook(…)`
  function is defined at the top as static Typst; the invocation block at the
  bottom is rendered by Askama with `{{ }}` / `{%- for %}` syntax. The file can
  also be compiled standalone with `typst compile` using the example data in the
  invocation block.
- `askama.toml` added to register `.typ` as a text-escaped extension.
- `render_typst` now just populates a `LogbookTemplate` struct and calls
  `.render()`; no string building or sentinel splitting.
- `POST /-/my/logbook.typ` endpoint added: returns the rendered Typst source as
  a download. A second button ("Download Typst source") added to the form via
  `formaction`.

**2026-04-04T21:51:28Z**

Logbook template and layout overhaul:

- Moved route from /-/export/logbook.pdf to /-/my/logbook.pdf (and
  /-/my/logbook.typ for source download)
- typst binary resolved from PATH; startup check runs typst --version and warns
  if missing

  logbook()

  checked/enabled by default
- Two submit buttons on the form: "Download PDF" and "Download Typst source"
  (formaction)
- Content-Disposition: filename only, no "attachment" directive
- Log page layout:
  - Landscape, DejaVu Sans 16 pt
  - Page header (1.5 cm): Date and Location write-in fields with equal spacing
    (h(1fr))
  - Page footer (1.5 cm): call sign left, page number (context counter) right
  - Header/footer only on log pages; optional pages use blank default margins
  - Table columns (fr units): QSO #, Station, Time, RST Sent, RST Rcvd, Freq,
    Mode, Power
  - Two rows per entry: 1.2 cm data row + full-width left/horizon Remarks row
  - Table vertically centered on page, fills full width
  - No gray fills (ink saving)
- Phonetic alphabet: compact grid (1 cm letter + 3 cm word) × 6 columns, no page
  break before it
- Frequency bands:
  - fmt-freq helper: MHz/GHz with narrow no-break space thousands separator
  - Full IARU Region 1 HF band plan with mode sub-allocations (CW, Digimodes,
    All modes, Beacons, Satellite)
  - VHF/UHF: 6 m, 2 m, 70 cm, 23 cm with CW/Digimodes/Voice sub-allocations
  - Nested table: band row + indented mode rows
