# Entity/contact model + baseline migration refactor

## Notes

**2026-01-28T12:05:27Z**

Design:

- Replace ham_club with a generic Contact + global Entity callsign registry.
- Entity: kind = repeater|contact; call_sign is optional but must be uppercase
  when present; unique across all entities.
- Contact: kind = organization|individual; holds display_name + contact fields.
- RepeaterSystem: keeps its own id; adds required FK to entity (1:1 via
  UNIQUE(entity)); replaces ham_club_id with optional owner and tech_contact
  (FKs to contact.id).
- Convention (not enforced by DB): entity.kind=repeater implies there is a
  repeater_system row linked via repeater_system.entity.

Acceptance:

- Baseline migration renamed from ham_club_baseline -> entity_baseline and
  defines new tables/constraints.
- Repeater baseline migration updated to new FKs/columns without adding a new
  migration.
- Codebase updated: no remaining ham_club references; Diesel schema/DAO updated.
- Full verification passes: make all, make test, make db-init, cargo run --bin
  load-data.
